# CVCL Tutorial

A step-by-step introduction to CVCL, runnable entirely from the terminal.
Each tutorial builds on the previous one. Run them in order.

---

## Prerequisites

Build the project first:

```bash
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
```

All tutorial binaries land in `build/examples/tutorial/`.

---

## Run All (Linux / macOS)

```bash
chmod +x run_all.sh
./run_all.sh
```

## Run All (Windows PowerShell)

```powershell
.\run_all.bat
```

---

## Tutorials

### 01 — Hello Image
**File:** `01_hello_image.c`

```bash
./build/examples/tutorial/01_hello_image
```

Covers:
- `cvcl_image_t` descriptor (width, height, channels, stride, depth)
- `cvcl_image_create` / `cvcl_image_free`
- `cvcl_fill` and `cvcl_set_u8`
- `cvcl_io_write_ppm`
- Why stride ≥ width×channels (64-byte SIMD alignment)

Output: `hello.ppm`

---

### 02 — Load and Inspect
**File:** `02_load_and_inspect.c`

```bash
./build/examples/tutorial/02_load_and_inspect hello.ppm
```

Covers:
- `cvcl_io_read_ppm`
- `cvcl_image_row` -- the right pattern for iterating pixels
- `cvcl_get_u8` for random access
- Zero-copy crop via `cvcl_image_view`
- Why you must NOT free a view

Output: prints pixel values and statistics

---

### 03 — Channel Conversion
**File:** `03_channels.c`

```bash
./build/examples/tutorial/03_channels hello.ppm
```

Covers:
- `cvcl_convert_channels` (RGB→Gray, Gray→RGB, RGB→RGBA)
- BT.601 luma formula
- How channel count affects stride

Output: `gray.pgm`, `back_to_rgb.ppm`, stride comparison

---

### 04 — Blur
**File:** `04_blur.c`

```bash
./build/examples/tutorial/04_blur hello.ppm
```

Covers:
- `cvcl_blur_box` -- O(w×h) sliding window, same cost for k=5 and k=63
- `cvcl_blur_gaussian` -- fixed-point Q15 separable, cost grows with k
- When to use each
- All border handling modes (`REPLICATE`, `ZERO`, `REFLECT`)

Output: `blur_box5.ppm`, `blur_box31.ppm`, `blur_gauss5.ppm`, `blur_gauss15.ppm`, border variants

---

### 05 — Edge Detection
**File:** `05_edges.c`

```bash
./build/examples/tutorial/05_edges hello.ppm
```

Covers:
- `cvcl_sobel_x` / `cvcl_sobel_y` -- integer Sobel, F32 output
- Gradient magnitude visualization
- `cvcl_canny` -- full pipeline (blur → Sobel → NMS → hysteresis)
- Threshold tuning (lo:hi ratio)

Output: `sobel_x.pgm`, `sobel_mag.pgm`, `edges_tight.pgm`, `edges_loose.pgm`, `edges_canny.pgm`

---

### 06 — Morphology
**File:** `06_morphology.c`

```bash
./build/examples/tutorial/06_morphology hello.ppm
```

Covers:
- `cvcl_erode` / `cvcl_dilate`
- `cvcl_morph_open` / `cvcl_morph_close`
- When to use open vs close
- Why large kernels cost the same as small ones (monotonic deque)

Output: `binary.pgm`, `eroded.pgm`, `dilated.pgm`, `opened.pgm`, `closed.pgm`, `eroded_large.pgm`

---

### 07 — Threshold & Histogram
**File:** `07_threshold.c`

```bash
./build/examples/tutorial/07_threshold hello.ppm
```

Covers:
- `cvcl_threshold` (BINARY, BINARY_INV, TRUNC, TOZERO modes)
- `cvcl_threshold` with `CVCL_THRESH_OTSU` (automatic)
- `cvcl_histogram` (per-channel bin counts)
- `cvcl_equalize_hist` (contrast enhancement via CDF)

Output: `thresh_binary.pgm`, `thresh_otsu.pgm`, `equalized.pgm`, more

---

### 08 — Custom Allocator
**File:** `08_custom_allocator.c`

```bash
./build/examples/tutorial/08_custom_allocator
```

Covers:
- `cvcl_allocator_t` interface (alloc, free, ctx)
- Arena allocator (bump pointer, O(1) alloc, reset-to-free)
- Counting allocator (track bytes in flight)
- Mixing allocators for different images
- Zero heap dependency for embedded targets

Output: prints allocator statistics

## Viewing Output Images

All tutorial outputs are saved as **PNG files** -- no conversion needed.
Open them directly in any image viewer.

```bash
# Linux
eog hello.png          # Eye of GNOME
xdg-open blur_box5.png # Default viewer

# macOS
open hello.png

# Windows
start hello.png
```

The tutorials load `hello.ppm` as input (generated by tutorial 01)
and save all results as `.png` files viewable without any tools.
